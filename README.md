## 功能特性

### 采暖炉（G55 / G56 / G58）

- **Climate 实体**：采暖控制，支持 5 种模式：
  - 待机（Standby）
  - 普通采暖（Normal Heating）
  - 节能采暖（Heating Energy Saving）
  - 户外模式（Heating Outdoor）
  - 快速采暖（Fast Heating）
- **Water Heater 实体**：生活热水温度控制（35–65°C）
- **传感器**：燃烧状态、燃气用量、采暖设定温度（普通/节能模式）
- **预约管理**：开关、模式选择、自定义时间段（text 实体）
- **能耗统计**：燃气用量、燃烧时长、点火次数

### 热水器（E51 / E65 / E66 / E75 / E76 / E86 / E88 / E89）

- **Water Heater 实体**：热水温度控制
- **传感器**：燃烧状态、燃气用量、热水设定温度
- **开关**：按摩水模式（E65/E75/E86/E88/E89）、循环保温（E86/E88/E89）
- **运行模式选择**：正常/冬季节能/自动/浓薄水
- **预约管理**：开关、模式选择、自定义时间段

### 热泵温控器（0F090011）

- **开关**：热泵主机开关
- **模式选择**：制热 / 制冷
- **温度设定（number 实体）**：室内温度、制冷设定温度、制热设定温度
- **传感器**：室内温度、运行状态、连接状态

### RTC-626 温控器（0F090004）

- **开关**：温控器开关
- **温度设定（number 实体）**：室内温度设定（5–35°C）
- **传感器**：室内实测温度、温度设定、故障码

### 净水软水器（0F070006）

- **开关**：立即再生
- **传感器**：工作状态、盐量、水硬度、再生次数、盐量低报警、出水流量、故障码

## 安装

### HACS（推荐）

1. 打开 Home Assistant 中的 [HACS](https://hacs.xyz/)。
2. 进入 **Integrations** > **Custom repositories**。
3. 添加本仓库地址 `https://github.com/palafin02back/rinnai_integration`，类别选 **Integration**。
4. 搜索 "Rinnai" 并安装。
5. 重启 Home Assistant。

### 手动安装

1. 将 `custom_components/rinnai` 目录复制到 HA 配置目录下的 `custom_components/`。
2. 重启 Home Assistant。

---

## 配置

1. 进入 **配置** > **设备与服务**。
2. 点击 **添加集成**，搜索 **Rinnai**。
3. 输入林内智家 App 的用户名和密码。

> 建议使用子账号，不要使用绑定设备的主账号，降低封号影响。

---

## 使用说明

### 预约自定义（采暖炉 / E系列热水器）

1. 在 `select.reservation_mode` 中选择 "自定义1" 或 "自定义2"。
2. 在对应的 `text` 实体中输入时间区间，格式：`00:00-08:00,18:00-23:00`。

### 温度设定（number 实体）

温控器和热泵配套温控器支持直接在 HA 中滑动设定目标温度，无需打开 APP。

## 故障排除

| 现象 | 可能原因 |
|---|---|
| 集成添加后无实体 | 设备型号不在支持列表，查看 HA 日志确认 `deviceType` |
| 实体状态 `unavailable` | 设备在林内 App 中离线，或 MQTT 连接失败 |
| 温度设定后立即还原 | 正常现象（乐观状态 10 秒内），等待设备回包确认 |
| MQTT 断连后不自动恢复 | 版本过旧，请更新到最新版本（已内置指数退避自动重连） |

## 贡献与支持

**本项目目前仅由作者本人持有 G56 采暖炉进行过实测。** 其他型号（G55/G58 采暖炉、全系列 E 系热水器、温控器、净水器）的配置均基于协议逆向和文档推断，**尚未经过真机验证**。如果你使用的是这些设备，非常欢迎通过 Issue 或 PR 反馈实际使用情况。

### 提交 Issue

遇到问题时，请在 Issue 中提供以下信息，方便定位：

1. **设备型号**：林内 App 中显示的型号名称
2. **deviceType / classID**：在 HA 日志或 App 设备详情中可以找到（格式类似 `0F06000C`）
3. **问题描述**：实体缺失 / 状态不对 / 操作无响应 等
4. **HA 日志**：开启调试日志后的相关输出

   在 `configuration.yaml` 中添加以下内容后重启 HA：
   ```yaml
   logger:
     default: warning
     logs:
       custom_components.rinnai: debug
   ```

   然后在 **设置 → 系统 → 日志** 中下载完整日志，或直接粘贴相关部分到 Issue。

### 适配新设备

如果你的设备型号不在支持列表中（集成添加后没有任何实体），你可以尝试自行适配并提交 PR：

1. 在 HA 日志中找到你设备的 `deviceType`（格式如 `0F06000C`）
2. 参考 `custom_components/rinnai/devices/0F06000C.json` 的结构
3. 创建对应的 `devices/<deviceType>.json` 文件
4. 提交 PR，并在描述中说明设备型号和测试结果

### 验证已支持设备

如果你拥有以下设备并能确认功能正常（或发现问题），欢迎在 Issue 中反馈，帮助完善配置：
